# frontend_exam

## 前端工程師考試題目

請使用 `axios` 或是 `fetch` 串接文章相關 API。

---

### 1. 文章列表 API

* **API 位置**：`/api/articles`
* **Query 參數說明**：
  * `type`: `0` = 全部, `1` = 最新消息, `2` = 熱門消息, `3` = 體育相關, `4` = 娛樂相關 (預設為 `0`)
  * `page`: 目前頁碼 (預設為 `1`)
  * `per_page`: 每頁顯示幾筆，僅支援 `10`, `20`, `30`, `50` (預設為 `10`)

* **回傳格式**：
  ```json
  {
    "data": {
      "count": "200",
      "data": [
        {
          "createdAt": "2026-07-01T14:30:00.000Z",
          "description": "這是第 1 篇測試文章的詳細內文...",
          "id": 1,
          "title": "【最新消息】這是第 1 篇測試文章",
          "type": 1
        }
      ]
    }
  }
  ```

---

### 2. 單一文章查詢 API

* **API 位置**：`/api/articles/{id}`
* **Path 參數說明**：
  * `id` (路徑參數): 欲查詢的文章 ID 數字
* **回傳格式**：
  ```json
  {
    "data": {
      "content1": "這是第 1 篇測試文章的額外內容一。",
      "content2": "這是第 1 篇測試文章的額外內容二。",
      "content3": "這是第 1 篇測試文章的額外內容三。",
      "createdAt": "2026-07-01T14:30:00.000Z",
      "description": "這是第 1 篇測試文章的詳細內文...",
      "id": 1,
      "title": "【最新消息】這是第 1 篇測試文章",
      "type": 1
    }
  }
  ```

---

### 📋 需求內容

1. **文章列表展示**：
   - 以卡片列出文章，包含標題、描述、分類與建立時間。
2. **分頁功能**：
   - 可透過分頁跟左右按鈕切換頁數。

---

### 🌟 加分內容

- **分類切換**：
  - 可透過選單切換分類，預設為「全部」 (0=全部, 1=最新消息, 2=熱門消息, 3=體育相關, 4=娛樂相關)。
- **每頁顯示筆數**：
  - 可選擇每頁顯示幾筆文章 (10, 20, 30, 50)。
- **元件化封裝**：
  - 使用組件封裝文章卡片，並透過 prop 傳入文章資料。
  - 使用組件封裝頁碼選擇器，並透過 `v-model` 或是 `props` / `emit` 進行頁碼切換。
- **詳細資訊**：
  - 點擊文章卡片可以跳轉文章詳細頁面或是顯示彈窗（API：`/api/articles/{id}`）。
- **SEO 與 UX**：
  - 文章資料的 SEO 相關設定。
  - UX 體驗優化。
